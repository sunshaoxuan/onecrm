# UX-01 OneCRM 交互与视觉设计规范

## 1. 核心视觉理念
OneCRM 的视觉表现应追求**高级感（Premium）**与**专业性（Professional）**的平衡。

### 1.1 大字引导原则 (Hero Typography)
- **场景**：客户详情展示页。
- **表现**：编码（Code）与名称（Name）应占据页面显著位置，字体大且醒目，作为用户操作的“定海神针”。
- **目的**：让用户在进入页面的一瞬间通过视觉引导明确当前工作对象。

### 1.2 非紧凑布局 (Spacious Layout)
- **间距**：大幅增加元素间的负空间（Whitespace），拒绝“密密麻麻”的表格。
- **排版**：优先使用卡片（Cards）、侧边抽屉（Drawers）或松散的列表项。
- **多页签**：通过 Tab 垂直或水平分割功能模块，确保单一时刻只显示当前任务的核心内容。

### 1.3 响应式等级 (Responsive Hierarchy)
- **PC 优先 (PC-First)**：作为 CRM 系统，PC/Web 端是最高优先级的生产力工具。所有功能应首先确保在宽屏下的极致效率。
- **阶梯式适配**：优先级顺序为 **PC/Web > 平板 (Tablet) > 手机 (Mobile)**。
- **自适应组件**：在小屏幕上，应通过合理折叠或隐藏次要信息，确保核心工作流（如搜索、关键数据展示）依然顺畅。

### 1.4 品牌标识 (Branding: OneCRM)
- **规范拼写**：始终使用 **OneCRM**，注意大小写。
- **LOGO 美学**：若在 LOGO 中使用大写字母表现，需确保 "ne" 部分的字号略小，以维持视觉上的平衡感与美观度。

## 2. 界面结构
### 2.1 灵活重构
- 页面结构应具备响应式与组件化的特点，支持根据业务重要性动态调整模块的顺序或显示比例。

### 2.2 现代化视觉元素
- **配色**：使用和谐的现代化配色方案（如深邃蓝、清新灰、柔和绿），避免高饱和度的原生颜色。
- **动态特效**：在 Tab 切换、列表悬停、卡片展开时加入平滑的微动画（Tween animations）。

### 2.3 动态多语言交互 (Dynamic I18n Interaction)
- **自愈式渲染**：当用户切换至翻译尚未完成的语言时，页面应先显示原文（或占位），并在后台翻译完成后通过视觉动效（如淡入加载）自动替换为目标语言词条。
- **翻译覆盖指示**：在语言切换菜单中显式提供“翻译进度条”或“已完全覆盖”的视觉标签，消除用户的不确定性。
- **词条优先级**：AI 优先翻译画面可视区域内的词条，确保首屏体验。

## 3. 组件级要求
- **搜索列表**：支持模糊搜索，结果展开过程中保持流畅。
- **组织架构树**：支持拖拽管理关系，节点清晰，连接线柔和。
- **文件管理**：文件卡片应显示缩略图或清晰的类型图标，汇总信息应以可视化图表或高亮标签形式呈现。
