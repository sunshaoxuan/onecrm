# PROD-01 OneCRM 核心功能与设计需求分析

## 1. 项目愿景
OneCRM 旨在打造一个**画面时尚、极致好用**的客户信息管理（CRM）与工作引导软件。它不仅仅是一个数据库，而是一个能够通过现代化视觉语言引导用户高效完成工作的生产力工具。

## 2. 视觉与交互准则 (Visual & UX Principles)
- **非紧凑型布局**：拒绝传统的密密麻麻表格排列。
- **时尚与现代化**：使用大字引导、卡片化设计、磨砂玻璃效果（Glassmorphism）及流畅的微动画。
- **高效可用**：核心信息（如客户编码、名称）以醒目的大字形式作为视觉引导。
- **多页签架构**：采用清晰的 Tab 切换，避免单页面信息过载。
- **灵活重构**：页面结构支持逻辑上的灵活调整，以适应不同业务场景的重心。

## 3. 功能模块分解

### 3.1 客户基础门户 (Customer Portal)
- **大字引导区**：每一个客户详情页由“客户编码”和“客户名称”作为巨幕式的头部引导。
- **搜索与列表**：带即时搜索功能的左侧/顶部客户导航列表。

### 3.2 档案信息管理 (Core Data Management)
| 类别 | 描述 | 特点 |
| :--- | :--- | :--- |
| **基础信息** | 编码、名称、简称 | 醒目，唯一标识 |
| **组织结构** | 部、课及其成员的树型关系 | 支持自由组织，支持**公共视图**与**私有视图**发布 |
| **产品实施** | 已实施产品名称、版本、日期 | 追踪生命周期 |
| **合约文档** | 关联多份合同文件 | 支持多文件预览/下载 |
| **客户化信息** | 针对该客户的特殊定制及汇总 | 多文件归档 |
| **数据/特殊应用** | 客户数据资料、特殊应用配置 | 多文件结构化存储 |

### 3.3 技术与运维套件 (Technical & IT Operations)
- **远程连接中心**：
    - 多版本、多平台的文字说明。
    - **结构化参数源**：存储可作为“一键连接”工具直接调用的参数数据。
- **备份中心**：外部备份文件链接的统一管理。
- **需求池**：收集与汇总客户反馈需求（多文件及汇总视图）。

### 3.4 业务责任矩阵 (Responsibility Matrix)
- 明确各业务维度的负责人：
    - 营业负责人
    - 实施负责人
    - 运维负责人
    - 其他自定义角色

## 4. 关键技术特性
- **数据结构化**：远程连接信息需从纯文本演变为可机器读取的参数源。
- **多维度视图**：组织结构支持权限控制下的视图发布（Public/Private）。
- **文件管理系统**：支持多处“多文件及各汇总”的功能块，需具备高效的文件关联与统计能力。
