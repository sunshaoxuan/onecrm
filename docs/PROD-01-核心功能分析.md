## 1. 项目愿景
OneCRM 旨在打造一个**AI 驱动、画面时尚、极致好用**的客户信息管理（CRM）与工作引导软件。它不仅仅是一个数据库，而是一个能够通过现代化视觉语言与后台智能代理（AI Agent）共同协作，引导用户高效完成工作的生产力工具。

## 2. 核心设计哲学
- **AI 深度驱动**：系统背景大量集成 AI 能力，不仅用于辅助决策，更深度参与系统自身的构建（如自动翻译、动态重构）。
- **动态多语言 (Dynamic I18n)**：
    - **默认支持**：初期默认提供日（JA）、中（ZH）、英（EN）三种语言。
    - **无限扩展**：当新增任一语言时，系统应能自动识别画面词条，按出现顺序实时送入 AI 翻译并更新字典。
    - **自愈式翻译**：未翻译内容在页面加载时自动触发翻译并刷新，直至语言完全覆盖。
- **非紧凑型布局**：拒绝传统的密密麻麻表格排列。
- **时尚与现代化**：使用大字引导、卡片化设计、磨砂玻璃效果（Glassmorphism）及流畅的微动画。
- **高效可用**：核心信息（如客户编码、名称）以醒目的大字形式作为视觉引导。
- **多页签架构**：采用清晰的 Tab 切换，避免单页面信息过载。
- **灵活重构**：页面结构支持逻辑上的灵活调整，以适应不同业务场景的重心。

## 3. 功能模块分解

### 3.1 客户基础门户 (Customer Portal)
- **大字引导区**：每一个客户详情页由“客户编码”和“客户名称”作为巨幕式的头部引导。
- **搜索与列表**：带即时搜索功能的左侧/顶部客户导航列表。

### 3.2 档案信息详细模块 (Detailed Data Modules)
每一个客户档案需包含以下多维度的信息，且多数支持“多文件管理”及“汇总分析”功能：

| 功能子项 | 详细描述 | 存储形式 |
| :--- | :--- | :--- |
| **基础信息** | 编码、名称、简称 | 文本/大字引导 |
| **组织结构** | 部、课及其成员，自由组织树型关系 | 树结构/多视图 |
| **已实施产品** | 产品名称、版本、实施日期 | 列表/日期 |
| **合约管理** | 关联多个合同文件 | 多文件 |
| **客户化信息** | 定制化开发信息及各维度汇总 | 多文件 + 汇总 |
| **数据资料** | 客户提供的数据资源 | 多文件 |
| **特殊应用信息**| 特定业务逻辑下的应用信息及汇总 | 多文件 + 汇总 |
| **远程连接信息**| 连接参数、说明文档、多版本文字描述 | **结构化参数** + 文档 |
| **客户需求** | 原始需求收集、反馈及汇总 | 多文件 + 汇总 |
| **备份文件链接**| 外部存储或归档系统的跳转链接 | 链接集 |
| **其他文件** | 不属于上述分类的补充资料及汇总 | 多文件 + 汇总 |
| **责任矩阵** | 明确营业、实施、运维等各阶段负责人 | 角色关联 |

## 4. 关键交互流程 (Key Interactions)
- **搜索引导**：通过左侧搜索列表快速定位，选中后右侧巨幕显示大字编码名称。
- **视图切换**：通过 Tab 页签在上述 12 个功能模块间无缝切换，保持界面简洁。
- **视图发布**：组织结构支持“私有编辑”后“公开发布”的发布流。

## 5. 关键技术特性
- **数据结构化**：远程连接信息需从纯文本演变为可机器读取的参数源。
- **多维度视图**：组织结构支持权限控制下的视图发布（Public/Private）。
- **文件管理系统**：支持多处“多文件及各汇总”的功能块，需具备高效的文件关联与统计能力。
